### File schemas

#### `{domain}_papers_marker.json`

Array of paper objects. Each paper has:

| Field | Type | Description |
|---|---|---|
| `pdf_filename` | string | e.g. `"42.pdf"` — stem is the `paper_id` used everywhere |
| `title` | string | Paper title (manually corrected where Docling failed) |
| `doi` | string | DOI string, e.g. `"10.1126/science.1254642"` |
| `sections` | array | List of `{level, heading, body}` section objects |
| `figures_folder` | string | Relative path to figures directory |
| `figure_count` | int | Number of extracted figures |
| `crossref_metadata` | object | Raw CrossRef API response |
| `docling_processed` | bool | Whether Docling was used for content extraction |
| `marker_processed` | bool | Whether Marker was used for section extraction |

Section objects: `{level: int, heading: str, body: str}`

#### `{domain}_nodes.csv`

Per-domain citation graph nodes. Column names differ slightly by domain:

| Column | aiml | neuromorphic / neuroscience |
|---|---|---|
| `id` | ✓ | ✓ |
| `doi` | ✓ | ✓ |
| `title` | ✓ | ✓ |
| `abstract` | ✓ | ✓ |
| `pdf_url` | ✓ | ✓ |
| `year` | ✓ (float) | — |
| `date` | — | ✓ (YYYY-MM-DD) |
| `authors` | — | ✓ (semicolon-separated) |
| `category` | — | ✓ |

#### `{domain}_edges.csv`

Directed citation edges: `source, target` — both are `id` values from the corresponding nodes file.

#### `unified_citation_network_nodes.csv`

Cross-domain node index: `global_id, field, local_id` — maps a global integer ID to `(field, local paper_id)`.

#### `unified_citation_network_edges.csv`

Cross-domain citation edges using `global_id` values: `source, target`.

#### `github_repos/code_index.jsonl`

Pre-built search index over all cloned repository source files. Generated by running `dataset-assembly/scripts/code/index_code_files.py`. One JSON object per line:

| Field | Type | Description |
|---|---|---|
| `code_id` | string | Unique identifier: `"{repo_type}/{owner}/{repo}/{rel_path}"` |
| `repo_name` | string | Full GitHub name, e.g. `"jeshraghian/snntorch"` |
| `repo_type` | string | Category: `"pytorch"`, `"hdl"`, `"fugu"`, or `"superneuro"` |
| `rel_path` | string | File path relative to repo root, e.g. `"snn/neurons.py"` |
| `language` | string | `"python"`, `"verilog"`, or `"vhdl"` |
| `stars` | int | GitHub star count at time of scrape |
| `topics` | array | GitHub topic tags |
| `description` | string | Repo description from GitHub metadata |
| `content` | string | Full file content (files >500 lines or >50 KB are excluded) |

Only `.py`, `.v`, `.sv`, `.vhd`, `.vhdl` files are indexed. Noise directories (`__pycache__`, `.git`, `build`, `dist`, etc.) are skipped.

## Procedure

**NOTE:** After all fetching procedures, dedups, and in the case of citation networks, self-loops and parallel edges, were ensured.

### Neuromorphic data

The initial set of neuromorphic computing publications were fetched from a union of these folders of pdfs:

- [Brad's neuromorphic papers](https://www.dropbox.com/home/knight-shared/Papers/Brads-one-hope-Neuromorphic-paper%20set)
- [Katie's neuromorphic papers](https://www.dropbox.com/home/knight-shared/Papers/Katies_neuromorphic-Paper-set)
- The MongoDB data from NeuKRAG

Then, the following procedure was run:
- Content, including titles, and figures were fetched using Docling, and titles were maually corrected for any malformed scrapes.
- DOIs of these papers were fetched from the titles using CrossRef API. The CrossRef API was also used to create a template json with metadata.
- A citation network was assembled using the OpenCitations API.
- The citation network was clustered using Leiden 0.0001 and the largest cluster was given. The low resolution was chosen as this is a 'noise removal' step similar to that of NeuKRAG's pipeline.
- The remaining pdfs had their sections converted to Markdown using Marker, and inserted into the json.

This resulted in 1745 papers.

### Neuroscience data

The intial set of neuroscience data was done by fetching all BiorXiv preprint data with a neuroscience tag using the BiorXiv API.

From this, the following procedure was done:
- No direct citations were found between this set and the neuromorphic papers. With this in consideration, around 200 common papers, cited by both the BiorXiv set and the neuromorphic set were added into this set, found using the OpenCitations API.
- From the existing set so far, the network was expanded two hops into the BiorXiv citation network, again using the OpenCitations API.
- All corresponding pdfs were downloaded. And any pdf that couldn't be downloaded due to paywall was discarded.
- Since discarding some pdfs can disconnect the citation network, the largest connected component in the citation network was preserved.
- The following marker papers were considered:

    1. Neuron Modeling

        - Hodgkin & Huxley (1952): A quantitative description of membrane current and its application to conduction and excitation in nerve
        - Izhikevich (2003): Simple model of spiking neurons
        - Gerstner & Kistler (2002): Spiking neuron models: Single neurons, populations, plasticity (This is a book)

    2. Learning rules

        - Bi & Poo (1998):Synaptic Plasticity & Learning
        - Song, Miller, Abbott (2000): Synaptic modifications in cultured hippocampal neurons: dependence on spike timing
        - Competitive Hebbian learning through spike-timing-dependent synaptic plasticity
        - Pfister & Gerstner (2006): Triplets of spikes in a model of spike timing–dependent plasticity

    3. Cortical organization
        - Mountcastle (1997): The columnar organization of the neocortex

- All papers but `Hodgkin & Huxley (1952)` and `Mountcastle (1997)` were in this set. As such, these were added manually, pdfs downloaded, and wired into the citation network.
- The CrossRef data was used to create a json template with all fetched paper metadata.
- Finally, Marker was used to scrape sections of the paper into markdown text, and was inserted into the json.

### AI/ML data

Initial AI/ML data was fetched from the following HuggingFace datasets:

- [ArXiv ML Papers](https://huggingface.co/datasets/CShorten/ML-ArXiv-Papers)
- [ICML 2021-2024 Papers](https://huggingface.co/datasets/AIM-Harvard/ICML-Accepted-Papers)

From here, the following procedure was done:
- A unioned list of dois, across all tables were fetched for these papers, using the CrossRef API.
- Using the OpenCitations API, common citations were fetched between this doi list and the other fields: neuromorphic and neuroscience.
- Using the SemanticScholar API (works better with ArXiv dois). The set was expanded by 2 hops, resulting in 13,874 papers. With this still being too large a set, leading to risk of fetches dominating over the other two fields, this was filtered down more.
- The union of the following node sets were considered as 'important nodes in the network':
    - The papers citing other fields
    - The 100 most cited nodes
    - The following papers:
        - Attention Is All You Need
        - Wide Residual Networks
        - Conditional Generative Adversarial Nets
- A Steiner Tree was computed across the important nodes set.
- The 13k node citation network was clustered Leiden 0.1, and the steiner tree nodeset was expanded with nodes belonging to the same clusters as them. This resolution was chosen as compared to resolutions 0.01, 0.001, 0.0001 and 0.5, this procedural step resulted in a network of between 1000-2000 nodes, which is comparable to the other two fields. 
- CrossRef metadata was fetched of the remaining nodes, creating a json template.
- The corresponding papers to these nodes were downloaded. This was completely doable with a shell script.
- Marker was used to fetch sectioned markdown data from the papers, and was subsequently inserted into the template.

**NOTE**: After all publication data was fetched, an overarching citation network was built so that the LLM could later on fetch cross-disciplinary citations/inspirations.

    - `ORNL/superneuromat`
    - `ORNL/superneuroabm`

    - `"Fugu spiking neural network"`

    - `"SuperNeuro neuromorphic"`
    - `"SuperNeuro spiking"`
    - `"superneuromat"`
    - `"superneuroabm"`
    - `"ORNL superneuro"`
- The repositories were organized into their respective `fugu` and `superneuro` parent directories.

#### Code indexing

After all repositories are cloned, a flat search index is built over all source files so the agent can retrieve code at query time without walking the full repo tree on every request.

```bash
python dataset-assembly/scripts/code/index_code_files.py
```

This walks all four repo type directories, reads each eligible source file, and writes one JSON record per file to `data/github_repos/code_index.jsonl`. The index is loaded automatically by `NeuroGraphStore` at server startup if the file exists. Re-run the script whenever repositories are updated or new ones are cloned.
